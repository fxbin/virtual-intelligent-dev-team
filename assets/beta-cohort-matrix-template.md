# Beta Cohort Matrix

| Round | Archetype | Sample size | Why this user matters | Primary task | Expected risk |
| --- | --- | --- | --- | --- | --- |
| round-0 | first-time novice | 2 | validates first-run comprehension | first-run journey | promise is unclear |
| round-0 | goal-driven power user | 2 | validates shortcut and efficiency paths | fastest happy path | advanced path mismatch |
| round-0 | skeptical evaluator | 1 | validates perceived credibility | value proposition review | trust and differentiation failure |
| round-1 | daily operator | 4 | validates repeated usage quality | core repeated task | friction accumulates over time |
| round-1 | edge-case breaker | 2 | validates failure handling | invalid or unusual states | hidden blockers surface |
| round-2 | returning user | 6 | validates state continuity | resume interrupted task | regression in saved state |
| round-2 | long-tail edge-case user | 4 | validates broader variance | mixed advanced flows | latent defects emerge |

## Notes

- Keep each round heterogeneous enough to surface divergent signals.
- Do not expand sample size before the previous round's blockers are explicitly closed or accepted.
